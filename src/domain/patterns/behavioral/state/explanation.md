# State

## 概要
State（状態）パターンは、オブジェクトの内部状態が変化した際、その振る舞いを変えるためのパターンです。状態ごとの振る舞いを別々のクラスにカプセル化し、if文やswitch文による条件分岐を排除します。状態遷移ロジックを各状態クラスに委譲することで、コードの保守性と拡張性を向上させます。

## 主な登場人物
- **State（状態）**: すべての状態に共通のインターフェースを宣言する
- **ConcreteState（具体的状態）**: 特定の状態における振る舞いを実装する
- **Context（文脈）**: 現在の状態を保持し、状態オブジェクトに処理を委譲する

## 使う場面
- 自動販売機の状態管理（投入待ち、商品選択、販売中、在庫切れ）
- 注文処理システムの状態遷移（受付、決済中、発送済み、完了）
- ゲームキャラクターの状態（立ち、走り、ジャンプ、攻撃）
- TCP接続の状態管理（接続待ち、接続済み、切断中）
