# Command

## 概要
Command（コマンド）パターンは、リクエストをオブジェクトとしてカプセル化するパターンです。これにより、リクエストの種類ごとに異なるクライアントを用意することなく、同じインターフェースで様々なリクエストを扱えるようになります。また、リクエストのキューイング、ログ記録、取り消し（undo）などの機能を簡単に実装できます。

## 主な登場人物
- **Command（コマンド）**: 操作を実行するためのインターフェースを宣言する
- **ConcreteCommand（具体的コマンド）**: Receiverとアクションの間のバインディングを実装する
- **Receiver（受信者）**: 実際のビジネスロジックを持つクラス
- **Invoker（呼び出し元）**: コマンドを実行する役割を持つ
- **Client（クライアント）**: ConcreteCommandオブジェクトを生成し、Receiverを設定する

## 使う場面
- GUIアプリケーションのメニューアクションやボタンクリック
- 操作の取り消し（undo）とやり直し（redo）機能
- マクロ機能の実装（複数のコマンドをまとめて実行）
- 非同期処理のキューイングとスケジューリング
